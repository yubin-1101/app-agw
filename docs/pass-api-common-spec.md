# PASS API 공통 규격

## 연동 개요

단말 Client는 HTTP 프로토콜로 VM G/W에 `POST` 요청을 전송한다.
VM G/W는 처리 결과를 HTTP Body에 JSON 또는 BSON 포맷으로 반환한다.

현재 AppGateway 모듈은 JSON 기반 요청/응답을 우선 구현한다.
BSON 처리 필요 여부는 API별 규격서 확인 필요.

## 도메인 정보

| 환경 | Domain |
| --- | --- |
| 개발 서버 | `https://210.217.178.117:19443` |
| 운영 서버 | `https://vmgwspam.nate.com:9443` |

예: `https://127.0.0.1:19443/external/pass/v1/getUserInfo`

## 사용자번호 암호화

`userID`는 MDN 값을 의미하며 AES 방식으로 암호화한다.
암호화 키는 별도 전달받아야 하며, 운영 키는 코드/문서에 직접 기록하지 않는다.

| plain | encrypt |
| --- | --- |
| `01099990009` | `S+nHEtRM8t50QyLJB16Jpg==` |

현재 코드에서는 `AesManager`가 AES 복호화를 담당한다.
실제 암호화 키, IV, padding, base64 처리 방식은 운영 규격서 기준으로 최종 확인 필요.

## 표시 형식

- Request URL: `https://(DOMAIN):(PORT)/(SERVICE_URL)`
- 예: `https://127.0.0.1:19443/external/pass/v1/getUserInfo`
- HTTP Type: `HTTP`
- HTTP Method: `POST`
- Character set: `UTF-8`
- Request Parameter: HTTP Body JSON
- Response Data: HTTP Body JSON
- 필수 여부: `M` 필수, `O` 조건부 필수

- JSON 데이터 인코딩은 UTF-8 처리를 기본으로 한다.
- JSON 데이터 타입은 `String`, `Number` 형태를 기본으로 한다.
- 배열 필드명은 명세서에서 `jsonArrayObjectName[index]` 형식으로 표현한다.
- 이중 배열은 `jsonArrayObjectName[index][index]` 형식으로 표현한다.
- Array 값은 모두 default 값을 가지도록 한다.

## 필수 Request Header

| Field | 입력 값 | 구분 |
| --- | --- | --- |
| Content-Type | `application/json` | M |
| Content-Length | POST Body 크기 | M |

## 서버 접속 에러

HTTP 서버 접속 에러는 HTTP/1.1 규격을 따른다.

- `100`: 계속 작업합니다.
- `101`: 프로토콜을 전환합니다.
- `200`: 클라이언트 요청이 성공했습니다.
- `400`: 요청이 잘못되었습니다.
- `401`: 허용되지 않은 Client, `messageId=SVC3002`, `text=Unauthorized Client`
- `403`: 접근이 금지되었습니다.
- `404`: 찾을 수 없습니다.
- `405`: HTTP method를 허용할 수 없습니다.
- `406`: 클라이언트가 요청된 MIME 형식을 받아들이지 않습니다.
- `407`: 프록시 인증이 필요합니다.
- `412`: 전제 조건이 실패했습니다.
- `413`: 요청 엔티티가 너무 큽니다.
- `414`: 요청 URI가 너무 깁니다.
- `415`: 지원되지 않는 미디어 유형입니다.
- `416`: 요청된 범위를 충족시킬 수 없습니다.
- `417`: 실행하지 못했습니다.
- `423`: 잠금 오류입니다.
- `500`: 내부 서버 오류입니다.
- `501`: 헤더 값이 구현되지 않은 구성을 지정합니다.
- `502`: 게이트웨이/프록시 역할 중 잘못된 응답을 받았습니다.
- `503`: 서비스를 사용할 수 없습니다. Traffic 과부하 `messageId=SVC3401`, 서비스 상태 OFF `messageId=SVC3402`, `text=Service Unavailable`
- `504`: 게이트웨이가 시간을 초과했습니다.
- `505`: HTTP 버전이 지원되지 않습니다.

## 서버 처리 결과 코드

서버 처리 결과 코드는 `PassResponseCode` 기준으로 관리한다.

| retCode | 설명 |
| --- | --- |
| 1000 | API 요청 처리가 성공 |
| 1001 | 사용자 정보 없음. 인증된 MDN으로 고객 정보가 없음 |
| 1002 | MMS 보관 주기 만기 |
| 1010 | uAMP 서버 연동 에러 |
| 1011 | DB 서버 연동 에러 |
| 1012 | V-SMSS서버 연동 에러 |
| 1013 | MMSC 서버 연동 에러 |
| 1014 | 미지원 기능 |
| 1015 | 인증키 생성 오류. 모바일 망 인증 시만 가능 |
| 1100 | Key 인증 처리 성공 |
| 1400 | 잘못된 파라미터 값 전달 |
| 1401 | IP인증 에러 |
| 1402 | Key 인증 실패 |
| 1403 | Key인증 실패 |
| 1410 | 이미지 복호화 에러 |
| 1411 | 삭제할 메세지가 없습니다. |
| 1500 | 처리 오류가 발생 하였습니다. 다시 시도 하여 주십시오 |
| 1510 | 복원 유효 기간 오류. 보관 기간이 지나 복원할 수 없습니다. |
| 1511 | 이미지 보관 기간 오류. 이미지 보관 기간이 지나 복원할 수 없습니다. |
| 1512 | 복원 메시지 오류. 복원할 메시지가 없습니다. |
| 1513 | 보관 기간이 지나 신고할 수 없습니다 |
| 1514 | 신고 할 메시지가 없습니다 |
| 1600 | 사용량이 많아 지연되고 있습니다. 잠시 후에 이용하여 주십시오. |
| 1610 | 스팸필터링 서비스 가입 후에 사용 가능합니다 |
| 1611 | 인증이 실패 되었습니다. 잠시 후 다시 시도 바랍니다. |
| 1612 | 인증이 실패 되었습니다. SKT 통신망 상태에서 재 접속 해주세요. |
| 1613 | SK텔레콤 가입자만 사용 가능합니다 |
| 1614 | 인증 정보가 불일치하여 실패 되었습니다. 잠시 후 다시 시도 바랍니다. |

`15XX`는 서버 내부 오류 성격이며 서버 전달 메시지를 팝업 처리 후 진행한다.
`16XX`는 서버 외부 오류 성격이며 서버 전달 메시지를 팝업 처리 후 종료한다.
